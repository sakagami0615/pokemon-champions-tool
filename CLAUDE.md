# CLAUDE.md

## プロジェクト概要

このプロジェクトは、Pokemon Championsに使用するための個人利用ツールです。このツールでは以下のような機能を実現します。

- ポケモンチャンピオンズの対戦前に、相手パーティをもとにAIが選出予想を生成し、各ポケモンの型情報（技・持ち物・特性・性格・個体値・能力ポイント）を表示する

## 重要ルール

### 1. コード構成

- 少数の大きなファイルより、多数の小さなファイルを優先する
- 高い凝集度、低い結合度を保つ
- 1ファイルあたり通常200〜400行、最大でも800行まで
- 種類別ではなく、機能/ドメインごとに整理する
- フロントエンドとバックエンドでDockerコンテナを分ける
- クリーンアーキテクチャを採用する


### 2. コードスタイル

- コード、コメント、ドキュメントに絵文字を使わない
- 常にイミュータブル（不変）にする - オブジェクトや配列を変更しない
- 本番コードに console.log を残さない
- try/catch を使った適切なエラーハンドリング
- Zod などを使った入力バリデーション

### 3. テスト

- TDD（テスト駆動開発）：最初にテストを書く
- カバレッジは最低80%
- ユーティリティにはユニットテスト
- APIには統合テスト
- 重要なフローにはE2Eテスト
- 動作はDockerコンテナ内で実施

### 4. セキュリティ

- シークレット情報をハードコードしない
- 機密データは環境変数で管理する
- すべてのユーザー入力を検証する
- クエリは必ずパラメータ化する
- CSRF対策を有効にする

## ディレクトリ構成

フロントエンドとバックエンドはフォルダを分ける。フロントエンド、バックエンドに関わらずフォルダ内の構成はクリーンアーキテクチャのディレクトリ構成に従うこと。

## 主要パターン

### APIレスポンス形式

```typescript
interface ApiResponse<T> {
  success: boolean
  data?: T
  error?: string
}
```

### エラーハンドリング

```typescript
try {
  const result = await operation()
  return { success: true, data: result }
} catch (error) {
  console.error('Operation failed:', error)
  return { success: false, error: 'ユーザー向けメッセージ' }
}
```

## 環境変数

| 変数名 | 必須 | 説明 |
|--------|------|------|
| ANTHROPIC_API_KEY | 必須 | Anthropic APIキー。https://console.anthropic.com/ で取得 |

ローカル開発時は `.env` ファイルを作成して設定する（`.env.example` 参照）。
`.env` は `.gitignore` に含まれており、リポジトリにコミットしないこと。
