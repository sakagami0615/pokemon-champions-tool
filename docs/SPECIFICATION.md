## 共通

### 前提条件

| 項目 | 内容 |
|------|------|
| 対象ゲーム | ポケモンチャンピオンズ |
| 対戦ルール | シングルバトル（3体選出） |

### UXに関して

- 対戦前の短時間で使えること
- 入力は可能な限り自動化（画像認識）
- 認識ミスはすぐ修正できること
- 出力は直感的に判断できる形

### 使用するデータに関して

本ツールは[ポケモンチャンピオンズの攻略サイト](https://gamewith.jp/pokemon-champions)に記載されているデータを使用して作成する。以下の内容を厳守して開発を行う。

- 本サイトに記載されている[GameWithの禁止事項](https://gamewith.jp/help/show/prohibited#gsc.tab=0)に抵触しない
- 本サイトのサーバー負荷をかけないようにする

### 設計方針

#### モジュール分割

処理は以下のように分離する

| モジュール | 内容 |
|------|------|
| 入力処理 | 画像 → ポケモン認識 |
| データ取得 | 使用率データの取得 |
| 選出推定 | AI / ルールベースなど |
| 出力整形 | 表示用データ生成 |

#### AIモデル選定

選出予想に使用するAIモデルは以下の2方式を候補とする

| 方式 | 具体例 | メリット | デメリット |
|------|------|------|------|
| 外部APIサービス | Claude API, ChatGPT API | 高性能・セットアップ不要 | API費用・インターネット接続が必要 |
| ローカルLLM | Ollama + Llama / Gemma など | 無料・オフライン動作 | 性能が劣る・マシンスペックが必要 |

初期実装は外部APIサービスを使用し、精度を検証する。必要に応じてローカルLLMへの切り替えを検討する。

#### 拡張性

将来的に以下を追加可能な設計とする

- 対戦中の再推定
- 構築分析機能
- ダメージ計算
- 型予測精度向上
- RAG連携

## 選出予想機能

### 目的

相手パーティをもとに、相手が選出してきそうなポケモン3体のパターンを予想し、選出判断の材料を提供する。

### 入力

#### 相手パーティ

- バトル前の選出画面を写真で撮影
- 画像認識によりポケモンを抽出する

#### 自分パーティ

- あらかじめアプリ内で登録しておく
    - パーティ登録用のUIを用意する(余裕があれば画像でポケモン登録できるようにしたい)
    - 複数パーティを登録できるようにする
    - 使用するパーティは簡単に切り替えられるようにする
- 選出支援時は自動で読み込まれる

#### 修正（画像認識失敗時）

画像認識に失敗した場合、以下の方法で修正可能とする

- キーボードによる手動入力
- 候補リスト（トグル形式）から選択

### 参照データ

#### タイプ相性 / 特性の説明

事前に用意したデータを使用する。

> 使用しない可能性があるので、優先度は低い

#### 内定ポケモン一覧

内定ポケモン一覧に関しては、[ポケモンチャンピオンズの攻略サイトの内定ポケモン](https://gamewith.jp/pokemon-champions/546414)に掲載されているデータを利用する。対象データは各ポケモンが持つ以下の情報となる。
なお、メガシンカするポケモンはメガシンカした後のデータも参照できるようにする。

| 種類 | 内容 |
| --- | --- |
| 図鑑ID | 図鑑に登録されている番号 |
| ポケモン名 | ポケモンの正式名称 |
| タイプ | ポケモンが持つタイプ |
| 種族値 | H/A/B/C/D/Sの種族値 |
| 身長/体重 | 身長(m)/体重(kg) |
| けたぐり・くさむすびの威力 | ポケモンの重さによって威力が変わるわざ(けたぐり・くさむすび)の威力 |
| 特性 | ポケモンが持つ特性の名前 |
| 弱点・タイプ相性 | 相性が良い(x0.5, x0.25)タイプと相性が悪い(x2, x4)タイプ |


#### 使用率ランキング

使用率ランキングに関しては、[ポケモンチャンピオンズの攻略サイトの使用率ランキング](https://gamewith.jp/pokemon-champions/555373)にランキングTOP30のポケモンの情報を使用する。

- ポケモンごとに別のページにバトルデータの詳細が記載されているため、そのページから各ポケモンのバトルデータ詳細を取得する。
- ポケモンの使用率はシーズンやレギュレーションによって変化するため、データを取得した日(正確に言うとGameWithページの最終更新日時)によって使用率が異なる。そのため取得したデータは取得した日ごとに管理できるようにする。

| 種類 | 内容 |
| --- | --- |
| データ集計日時 | ツールでデータを取得した日時 |
| シーズン番号 | 取得したデータのシーズン番号 |
| レギュレーション | 取得したデータのレギュレーション |
| rawデータ最終更新日時 | GameWithページの最終更新日時 |
| ポケモン名 | ポケモンの正式名称 |
| わざ | わざ名と使用率 (%) |
| 持ち物 | 持ち物名と使用率 (%) |
| 特性 | 特性名と使用率 (%) |
| 性格 | 性格名と使用率 (%) |
| チーム | 同じパーティとして採用されるポケモンと使用率 (%) |
| 能力 | H/A/B/C/D/Sの能力ポイントと使用率 (%) |

> 取得できそうなものは取得しているため、上記の表のうち使用しないデータが出てくるかもしれない。

ランキングTOP30外のポケモンに関しては、内定ポケモン一覧に記載した情報をもとに上記の表の項目を埋める形で補間する。
補間できないデータに関しては、「ランキング外」で補間する。

### 処理内容（※要検討領域）

1. 相手パーティ（6体）を入力として受け取る
2. 使用率データを参照する
3. 自分パーティと相手パーティと使用率データをAIのコンテキストとして与えて相手の選出3体を予想する
4. 選出予想は複数（約3パターン）生成する

> 選出されるポケモンの正解データはないため、YouTubeの動画などが使用できないか検討する必要がある。

### 出力　

#### 選出予想

- AIによる選出予想パターンを3パターン程度表示する
- パターンは選出されそうな可能性が高い順に並べる
- 各パターンの確率（%）は表示しない

#### 各ポケモンの型情報

各選出パターンに含まれるポケモンについて、以下を表示する。

- 技の候補と使用率（%）
- 性格の候補と使用率（%）
- 持ち物の候補と使用率（%）
- 特性の候補と使用率（%）
- 能力の候補と使用率（%）

> 各項目ごとに複数候補が存在する。どの程度表示するかは実装しながら検討する。

## パーティ分析機能

### 目的

- 自身のパーティを入力し、どの相手に弱そうかなどの情報を分析する
- 仮想的パーティを入力し、仮想バトルを行わせて勝率がどの程度なのかを測れるようにしたい
