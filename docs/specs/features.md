# 機能仕様

## 0. データ未取得時の制約

スクレイピングで内定ポケモン一覧と使用率データは同時に取得されるため、通常は両方が揃った状態になる。いずれか一方でも欠損している場合は「データが存在しない」と見なし、以下の制約を適用する:
- 「選出予想」「自分パーティ登録」機能は使用不可

## 1. 相手パーティ入力

- 内定ポケモン一覧からポケモンを最大6体選択する
- 撮影した選出画面の写真をアップロードすると、OpenCV テンプレートマッチングで6体を自動認識して入力できる
- 認識結果は手動で修正可能

## 2. 選出予想

**実行条件**:
- 相手パーティに3体以上のポケモンが入力されている
- 自分のパーティが選択済み

**処理**:
- 相手パーティ6体・自分パーティ・使用率データ（選択中の日付。未選択の場合は最新）を Claude API に渡す
- 3パターンの選出予想を返させる

**出力データ（各ポケモンカードに表示する項目）**:
- スプライト画像 ＋ ポケモン名
- わざ（上位3件 ＋ 使用率%）
- 持ち物（上位2件 ＋ 使用率%）
- 特性（上位2件 ＋ 使用率%）
- 性格（上位2件 ＋ 使用率%）
- 種族値（H/A/B/C/D/S）
- 能力ポイント（上位2件のEV配分 ＋ 使用率%）

## 3. 自分パーティ登録

**保存条件**:
- パーティ名が1文字以上入力されている
- ポケモンが3体以上選択されている

- 複数パーティを上限なしで登録・保存できる
- パーティ名をつけて切り替え可能
- 各パーティに内定ポケモン一覧からポケモンを最大6体登録する
- 選出予想画面を開いたとき、前回使用したパーティを自動で読み込む

## 4. データ管理

### データ取得

- 「データ取得」ボタン押下で GameWith スクレイピングをバックグラウンドで開始する（UI は操作可能なまま）
- スクレイピング完了後、新しい日付のデータが利用可能になる

### データ選択

- 取得済みデータの日付を選択することで、選出予想に使用するデータを切り替えられる
- 選択した日付はバックエンドのインメモリで保持（コンテナ再起動でリセット）

### バックエンド API

| エンドポイント | 説明 |
|---|---|
| `GET /api/data/status` | スクレイピング状態・選択日付・利用可能日付一覧を返す |
| `POST /api/data/fetch` | スクレイピングをバックグラウンド開始 |
| `GET /api/data/dates` | 利用可能な日付一覧を返す |
| `POST /api/data/select-date` | 使用する日付を設定する |
| `GET /api/data/pokemon-list` | 現在選択中の日付の内定ポケモン一覧を返す（使用率順にソート済み） |

### スクレイピング対象

- GameWith の内定ポケモン一覧ページ → スプライト画像 ＋ ポケモン基本情報
- GameWith の使用率ランキング TOP30 ページ → 各ポケモンの詳細バトルデータ
- 取得データは `data/usage_rates/YYYY-MM-DD.json` として日付管理
- TOP30 外のポケモンは内定ポケモン一覧データで補間、使用率は「ランキング外」とする

## 5. モデル設定

ヘッダーの「設定」タブ内「モデル設定」セクションから設定する。

- 対応プロバイダー: Anthropic / OpenAI / Google / Ollama（ローカルLLM）
- プロバイダーとモデルをラジオボタン＋プルダウンで選択する
- 各プロバイダーの APIキーは設定画面から入力・保存する
- Ollama はエンドポイントURL（デフォルト: `http://host.docker.internal:11434`）を指定し、「一覧を取得」ボタンでインストール済みモデルを取得する
- 選択中プロバイダーのモデルが未設定の場合、保存ボタンを無効化する
- 設定は `data/llm_config.json` に永続化される

### バックエンド API

| エンドポイント | 説明 |
|---|---|
| `GET /api/llm-config` | 現在の LLM 設定（プロバイダー・モデル・APIキー）を返す |
| `PUT /api/llm-config` | LLM 設定を保存する |
| `GET /api/ollama-models` | 指定エンドポイントのOllamaインストール済みモデル一覧を返す |
