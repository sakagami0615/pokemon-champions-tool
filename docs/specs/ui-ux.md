# UI / UX 設計

## レスポンシブレイアウト

**横長（PC・横向きスマホ）**:
- ポケモンカード内: ポケモン画像 左 ／ 型情報 右

**縦長（スマホ縦向き）**:
- ポケモンカード内: ポケモン画像 上 ／ 型情報 下

## ダークモード

- ヘッダーに手動トグルボタンを配置
- Tailwind CSS の `dark:` クラスで実装
- 設定は localStorage に保存（リロード後も維持）

## UX 原則

- 対戦前の短時間で使えること
- 認識ミスはすぐ修正できること（入力枠を直接編集）
- 出力は直感的に判断できる形（全パターン常時展開）

## データ未取得時

- 「選出予想」「自分パーティ登録」のボタンを無効化する
- 画面上に「データを取得してください」などのメッセージを表示し、データ管理ページへのリンクを添える

## 相手パーティ入力画面

- 6枠のスロットをデフォルト表示する
- 各スロットはプルダウンで内定ポケモン一覧からポケモンを選択する
- 選択後、スロット内にスプライト画像＋ポケモン名を表示する
- 画像アップロードボタンを配置し、撮影した選出画面の写真から自動認識できる

## 選出予想画面

- 選出予想の結果は3パターンを常時全展開で表示する
- 各パターン内に3体のポケモンカードを横並びで表示する
- 各ポケモンカードに以下を表示:
  - スプライト画像 ＋ ポケモン名
  - わざ（上位3件 ＋ 使用率%）
  - 持ち物（上位2件 ＋ 使用率%）
  - 特性（上位2件 ＋ 使用率%）
  - 性格（上位2件 ＋ 使用率%）
  - 種族値（H/A/B/C/D/S）
  - 能力ポイント（上位2件のEV配分 ＋ 使用率%）

## 自分パーティ登録画面

- 登録済みパーティの一覧を表示する
- 各パーティには、パーティ名と含まれるポケモンのスプライト画像＋ポケモン名を表示する
- ポケモン登録時はプルダウンで内定ポケモン一覧から選択する

## データ管理画面

ヘッダーの「データ管理」タブからアクセスする。

**レイアウト**:
- 左ペイン: 取得済み日付の一覧をラジオボタン付きで縦に並べる（各行に日付・内定ポケモン数を表示）
- 中央ペイン: 左ペインで選択した日付の内定ポケモン一覧をパネル形式で表示（各パネルにスプライト画像＋ポケモン名）
  - 表示順: 使用率ランキングがあるポケモンを使用率の高い順に先頭に並べ、ランキング外のポケモンはその後ろにポケモンNo順で並べる

**スクレイピング中の表示**:
- スクレイピング中はインジケーターを表示する
- 完了後、新しい日付のデータが左ペインのリストに追加される
